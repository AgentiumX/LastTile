# Last Tile（最后一格）

## 1. 项目概述

### 一句话描述

**走过即消失的格子迷宫——每一步都是不可逆的选择。**

### 产品定位

极简策略解谜单机手游。

### 核心体验

玩家面对一张格子地图，从起点走向终点。走过的格子永久消失，不能回头。必须在有限的步数内找到唯一正确的路径。

**产品不依赖：**
- 社交、排行榜、抽卡、数值养成、剧情

**核心驱动力来自：**
- 不可逆决策的紧张感
- "就差一点"的未完成感（Near Miss）
- 路线优化的成就感
- 发现隐藏路径的惊喜感

---

## 2. 为什么好玩

### 好玩的本质：选择 + 后果

一个解谜游戏好不好玩，取决于一个公式：

> **趣味 = 有意义的选择 × 不可逆的后果**

- 没有选择 → 无聊（两条直线到终点）
- 有选择但可撤销 → 不紧张（普通迷宫）
- 有选择且不可撤销 → 紧张 + 刺激 + 上瘾

Last Tile 的坍塌机制让每一步都不可撤销。这就是它区别于所有普通迷宫游戏的地方。

### 四层乐趣

| 层次 | 体验 | 举例 |
|------|------|------|
| 第1层：找到路 | "我知道怎么到终点" | 看到一条绕过墙壁的路径 |
| 第2层：走对路 | "我成功走过去了" | 在步数限制内到达终点 |
| 第3层：走最优路 | "我找到了最短路径" | 用更少步数通关，获得更多星 |
| 第4层：发现隐藏路 | "原来可以这样走！" | 发现传送门捷径、钥匙的巧妙路线 |

**大多数玩家卡在第2层和第3层之间，这正是"就差一点"的甜蜜区。**

### 反面教材：为什么当前版本不好玩

当前版本的问题：**地图上没有墙壁，从起点到终点只需走两条直线。**

```
S □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □
□ □ □ □ □ □ □ □ E
```

- 没有墙壁 → 没有障碍 → 没有绕路
- 没有绕路 → 没有选择 → 没有策略
- 没有策略 → 不好玩

**修复：墙壁是策略的根基。没有墙壁就没有游戏。**

## 3. 核心机制

### 3.1 地图

地图由格子组成，包含墙壁。墙壁不可进入。

```
S □ □ █ □ □ □ □
□ █ □ █ □ █ □ □
□ █ □ □ □ █ □ □
█ □ □ █ □ □ █ □
□ □ █ □ □ █ □ □
□ █ □ □ █ □ □ □
□ □ □ █ □ □ █ □
□ □ □ □ █ □ □ E
```

- S：起点
- E：终点
- █：墙壁（不可进入）
- □：可通行格

### 3.2 坍塌机制（核心中的核心）

**玩家离开格子后，该格永久消失，不可再次进入。**

```
移动前：        移动后：
S □ □          ■ P □
□ □ □          ■ □ □
```

■ = 已坍塌，不可进入。P = 玩家当前位置。

坍塌是 Last Tile 的灵魂。它让每一步都变成不可逆的承诺：
- 你不能回头换路
- 你不能试探后撤回
- 你走过的路就是你的命运

### 3.3 核心矛盾

**墙壁迫使绕路，坍塌禁止回头。**

这就是 Last Tile 的全部策略深度：

1. 墙壁挡住直线路径 → 你必须绕路
2. 绕路消耗更多步数 → 步数有限
3. 走过的格子消失 → 你不能回头换路
4. 选错路 = 被困或步数耗尽

**没有墙壁，就没有选择。没有选择，就没有策略。没有策略，就不好玩。**

### 3.4 移动规则

- **允许：** 上、下、左、右（4方向）
- **不允许：** 斜向移动
- **不可进入：** 墙壁、已坍塌格、地图边界外

### 3.5 失败条件

1. **步数耗尽** — 没走到终点但步数用完
2. **无路可走** — 四面都是墙壁/坍塌格/边界
3. **被困死** — 还有路走，但永远到不了终点

被困死示例：
```
■ ■ █
■ P ■
█ □ ■
```
玩家下方有路，但走过去也到不了终点。

### 3.6 胜利条件

到达终点。

## 4. 地图元素

### 4.1 墙壁 █（核心）

不可进入。迫使玩家绕路，是一切策略的基础。

**墙壁的设计原则：**
- 墙壁必须创造**选择点**——走到这里，必须决定往哪边绕
- 墙壁必须创造**死路**——看起来能走，但走进去就出不来
- 墙壁不能创造**唯一路径**——如果只有一条路，那不叫策略，叫走迷宫

好的墙壁设计：
```
S □ █ □
□ □ █ □
□ □ □ □    ← 可以从下方绕过，也可以从右侧绕过
█ □ █ □    ← 两条路，选哪条？
□ □ █ E
```

坏的墙壁设计：
```
S □ █ □ □ □ E
□ □ █ □ □ □ □
□ □ █ □ □ □ □
```
只有一条路绕过墙壁，没有选择。

### 4.2 普通格 □

无效果，可通行。走过后坍塌。

### 4.3 传送格 A↔A

进入后瞬移到配对格子。创造空间跳跃，让路线规划突破物理距离。

**设计要点：** 传送门是"捷径"也是"陷阱"——跳过去后，你原来的路就断了（坍塌），回不来了。

### 4.4 钥匙格 🔑

踩中后获得一把钥匙。创造路线优先级——你必须先去拿钥匙，再去开锁。

**设计要点：** 钥匙的位置决定了路线的形状。钥匙放在"绕路"的位置上，就迫使玩家在"先拿钥匙再走"和"直接走但开不了锁"之间选择。

### 4.5 锁格 🔒

必须有钥匙才能进入。锁挡住的路是"条件性通路"——有钥匙就能走，没钥匙就是墙。

**设计要点：** 锁放在关键通道上，迫使玩家必须先完成"拿钥匙"这个子目标。

### 4.6 桥梁格 ═

踩中后在指定方向生成新路径。创造"先激活再通过"的时序谜题。

### 4.7 翻转格 ⟳

触发后地图局部重建（如消失区域恢复）。创造"二次机会"的惊喜。

## 5. 关卡设计圣经

### 5.1 关卡设计铁律

**铁律1：每一关必须有至少一个选择点**

选择点 = 玩家必须决定"往左还是往右"的位置。没有选择点的关卡不合格。

**铁律2：最优路径不能是曼哈顿距离**

如果最优步数 = |Δrow| + |Δcol|，说明没有绕路，说明墙壁没起作用，说明关卡设计失败。

**铁律3：必须存在至少一条"诱导错误路线"**

看起来能走，但走过去要么是死路，要么浪费太多步数。这是"就差一点"体验的来源。

**铁律4：前3关不使用任何特殊格**

只用墙壁 + 坍塌。让玩家先理解核心机制，再引入新元素。

**铁律5：每个新元素单独教学一关后再组合**

传送门、钥匙锁、桥梁——每个元素先在简单关卡中单独出现，让玩家理解后再组合使用。

### 5.2 好的关卡模式

**模式1：岔路（The Fork）**

```
S □ □
  █ █
  □ □ E
```
两条路，一条更短。玩家必须判断哪条更优。

**模式2：诱饵（The Bait）**

```
S □ □ □
  █ █ □
  □ □ █ E
```
看起来直走最近，但直走是死路。必须绕远路。

**模式3：钥匙绕路（The Key Detour）**

```
S □ █ □ E
  □ █ □ █
  K □ □ L
```
锁挡住了直达终点的路，必须先绕到下方拿钥匙。

**模式4：瓶颈（The Bottleneck）**

```
S □ █ □ □
  █ □ █ □
  □ □ □ □ E
```
中间只有一个通道。走错方向穿过通道，就再也回不来了。

**模式5：蛇形（The Serpent）**

```
S □ █ █ █ █ █
  □ █ □ □ □ □ █
  □ □ □ █ █ □ █
  █ █ □ □ □ □ E
```
必须按蛇形路线前进，坍塌让回头不可能，每一步都必须精确。

### 5.3 坏的关卡模式（绝对禁止）

**反模式1：直线（The Straight Line）**

```
S □ □ □ □ □ □ E
```
没有选择，没有策略。**这就是当前版本的问题。**

**反模式2：唯一路径（The Single Path）**

```
S □ █ □ █ □ E
  █ □ █ □ █ □
```
虽然有墙壁，但只有一条路可走。等于没有选择。

**反模式3：纯猜谜（The Guessing Game）**

完全看不到哪条路对，只能盲猜。玩家不是在思考，而是在赌博。

**反模式4：马拉松（The Marathon）**

路径太长，步数太多，玩家感到的是疲惫而非挑战。

### 5.4 关卡验证标准

每一关上线前必须通过以下检查：

| 检查项 | 标准 |
|--------|------|
| 可解性 | A*求解器必须能找到解 |
| 非平凡性 | 最优步数 > 曼哈顿距离（说明有绕路） |
| 选择性 | 至少存在2条不同的可达路径 |
| 诱导性 | 至少存在1条"看起来对但走不通"的路径 |
| 步数合理性 | 容错步数 ≥ 3（新手关）到 ≥ 1（专家关） |
| 单局时长 | 最优解 ≤ 30步（保证90秒内完成） |

## 6. 步数系统

### 基础逻辑

每关存在理论最优步数。系统给玩家的可用步数 = 最优步数 + 容错步数。

### 难度曲线

| 难度 | 容错步数 | 对应关卡 |
|------|----------|----------|
| 新手 | 最优+8步 | 第1-5关 |
| 入门 | 最优+5步 | 第6-15关 |
| 普通 | 最优+3步 | 第16-30关 |
| 困难 | 最优+2步 | 第31-45关 |
| 专家 | 最优+1步 | 第46-50关 |

## 7. 关卡进度设计

### 教学阶段（第1-5关）

**目标：** 让玩家理解"坍塌"和"墙壁"两个核心概念。

| 关卡 | 地图 | 新元素 | 教学目标 |
|------|------|--------|----------|
| 1 | 3×3 | 无 | 学会移动：从S走到E |
| 2 | 3×3 | 墙壁 | 学会绕路：一面墙挡住直路 |
| 3 | 3×4 | 墙壁 | 学会规划：多面墙，必须想好再走 |
| 4 | 4×4 | 墙壁+死路 | 学会识别陷阱：看起来能走但走不通 |
| 5 | 4×4 | 墙壁+坍塌 | 体验坍塌：走错一步就回不去了 |

### 入门阶段（第6-15关）

**目标：** 引入特殊格，扩展策略维度。

| 关卡 | 新元素 | 教学目标 |
|------|--------|----------|
| 6-7 | 钥匙+锁 | 先拿钥匙再开锁 |
| 8-9 | 传送门 | 利用传送门跳过障碍 |
| 10 | 钥匙锁+墙壁 | 组合使用已学元素 |
| 11-15 | 渐进复杂 | 更大的地图，更多的墙壁 |

### 普通阶段（第16-30关）

**目标：** 多元素组合，步数收紧，需要精确规划。

- 6×6 到 7×7 地图
- 钥匙锁 + 传送门 + 墙壁组合
- 容错步数减少到+3
- 出现多钥匙/多锁的关卡

### 困难阶段（第31-45关）

**目标：** 紧凑步数，复杂路线，需要多次尝试。

- 7×7 到 8×8 地图
- 所有元素自由组合
- 容错步数减少到+2
- 出现双传送门对、多钥匙锁链

### 专家阶段（第46-50关）

**目标：** 极限挑战，每一步都必须完美。

- 8×8 地图
- 复杂元素组合
- 容错步数仅+1
- 最优路线隐藏很深

## 8. 核心反馈系统

### 失败反馈

**禁止：**
```
失败
```

**改为：**
- `距离终点：剩余3格`
- `你发现了84%的地图路线`
- `最优路线：26步，当前最佳：29步`

**重点：** 永远让玩家感觉进步。

### 坍塌反馈

格子消失时必须有明确的视觉+音效反馈，让玩家意识到"我走过的路没了"。

- 视觉：格子碎裂/下沉动画
- 音效：轻微碎裂声

### 接近终点反馈

- 距终点3格内：心跳加速音
- 只差1格失败：短暂停顿 + 轻微回响

## 10. 未完成感设计

通关后显示：
```
关卡完成
★★★★☆
```

**评分条件：**
- 最优步数 = 5星
- 最优+1步 = 4星
- 最优+2步 = 3星
- 最优+3步 = 2星
- 更差 = 1星

玩家获得4星 → "再来一次，我肯定能5星"

## 11. 视觉设计

**目标：** 极简

**参考：** 极简几何风、黑白+一种强调色

**元素区分：**
- 可通行格：浅灰
- 墙壁：深黑+边框（明显区分于可通行格）
- 已坍塌格：深黑+X标记
- 起点：绿色
- 终点：红色/橙色
- 玩家：强调色（如蓝色）
- 钥匙：金色
- 锁：暗金色
- 传送门：紫色

## 核心成功指标（North Star Metric）

不是日活，不是付费率，而是**单关平均重试次数**。

**目标：**
- 新手关：1.5~2次
- 普通关：4~8次
- 精英关：10~20次

因为《Last Tile》的本质不是让玩家不断前进，而是让玩家持续处于：

> "我已经看见答案了，只差最后一步。"

的心理状态。
